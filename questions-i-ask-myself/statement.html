Starting out with an empty sheet of paper, I start listing down random integers in increasing order. Every now and then, I stop to think. I look at part of my list and ask myself: "How many of these integers have as a divisor?" I realized that I waste a lot of time doing this repeatedly and I wish to automate the process.

Input Format
The first line of input contains , the number of queries.

The following lines contain the queries. Each query is either ADD V or ASK A B M on a single line.

  • ADD V. This adds the integer at the end of our list. It is guaranteed that integers are added in increasing order.
  • ASK A B M. For every query of this type, you need to output the number of integers currently in our list such that is at least , at most and has as its divisor.

Output Format
For each ASK query, output a line containing the number of integers currently on the list such that is at least , at most and has as its divisor.

Constraints




The integers are added in increasing order.

Sample Input 

10
ADD 1
ADD 2
ADD 3
ADD 5
ASK 2 8 2
ADD 8
ASK 2 8 2
ADD 13
ADD 21
ASK 1 300000 9

Sample Output 

1
2
0

Explanation
During the first ASK query, the list contains [1, 2, 3, 5]. There is only one element in this list which is at least , at most and has as its factor - this is the element 2.

During the second ASK query, the list contains [1, 2, 3, 5, 8]. There are only two elements in this list which are at least , at most and have as its factor - these are the elements 2 and 8.

During the third ASK query, the list contains [1, 2, 3, 5, 8, 13, 21]. There are no elements in this list which are at least , at most and have as its factor.
